Status bar

A full-width sticky bar showing contextual key/value information. Used to keep participant or appointment context visible while scrolling through a long page.

Macro: appStatusBar(params)

In production usage, the status bar is rendered via the pageNavigation block so it appears above the main content and sticks to the top of the viewport on scroll. The examples below are rendered inline for demonstration.

The bar accepts one or more rows, each containing an array of items. Items can have a key/value pair, a plain text, or raw html.

Single row

{{ appStatusBar({ classes: "", rows: [ { items: [ { key: "Participant:", value: "Janet Williams" }, { key: "DOB:", value: "12 March 1965 (61)" }, { key: "NHS number:", value: "485 777 3456" } ] } ] }) }} 
{% raw %}{{ appStatusBar({
  rows: [
    {
      items: [
        { key: "Participant:", value: "Janet Williams" },
        { key: "DOB:", value: "12 March 1965 (61)" },
        { key: "NHS number:", value: "485 777 3456" }
      ]
    }
  ]
}) }}{% endraw %}

Two rows

Use two rows to separate appointment-level from participant-level information.

{{ appStatusBar({ classes: "", rows: [ { items: [ { key: "Clinic:", value: "Morning clinic, 31 March 2026" }, { key: "Unit:", value: "St Thomas' Hospital BSU" }, { key: "Status:", value: "In progress" } ] }, { items: [ { key: "Participant:", value: "Janet Williams" }, { key: "DOB:", value: "12 March 1965 (61)" }, { key: "NHS number:", value: "485 777 3456" } ] } ] }) }} 
{% raw %}{{ appStatusBar({
  rows: [
    { items: [...] },
    { items: [...] }
  ]
}) }}{% endraw %}

With HTML content

Use html on an item for rich content such as status tags.

{{ appStatusBar({ classes: "", rows: [ { items: [ { key: "Participant:", value: "Janet Williams" }, { html: "Checked in" } ] } ] }) }} 
{% raw %}{ html: "Checked in" }{% endraw %}
